JSONP

发表于 2018-06-30 | 更新于 2018-07-31 | 分类于 前端
初步了解 JSONP

什么是 JSONP（JSON with Padding）？

  1. 客户端通过动态创建 script 标签，使 script.src 指向请求的目标 URL ，并在 URL 后追加 callback 查询参数（随机数），来发起 GET 请求，以获取服务器的数据，并定义一个与 callback 参数（随机数）同名的函数来接收数据。
  2. 服务器接收到请求后，将数据放在 callback 函数，返回响应。
  3. 响应主体会被写进 script.src 指向的脚本文件，该文件里的内容会被当作 JS 代码执行，从而实现调用函数，浏览器可在 callback 函数里处理所传入的数据。

JSONP 为什么不支持 POST？

  1. JSONP 是通过动态创建 script 标签实现的
  2. 动态创建的 script 只能发起 GET 请求
  3. 动态创建的 script 的 src 只能写入 GET 请求的数据，不能写入 POST 数据

为什么 JSONP 没有跨域限制？

通过 <script> 标签的 src 属性，放入将要访问的目标 URL + callback 查询参数。浏览器遇到 <script> 标签时，会根据 src 属性的值向目标 URL 发起一次 GET 请求。

比较 JSONP 与 Ajax 请求

  • Ajax 请求可以读取服务器返回的响应（可能包含有 DOM、Cookie 等），因此受同源政策限制。
  • 与 Ajax 请求不同的是，JSONP 并不能去读取服务器返回的响应。JSONP 只是以类似于请求资源的方式去向服务器请求数据，服务器会将数据作为参数放到原查询参数 callback 里，然后返回给浏览器，故不受同源政策的限制。对于浏览器来说，就是获取到了一个脚本资源而已。浏览器会去执行该脚本，会去调用预先定义好的 callback 函数，而参数就是请求的数据。
