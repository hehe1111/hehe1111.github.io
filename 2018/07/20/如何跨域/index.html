如何跨域

发表于 2018-07-20 | 更新于 2018-07-30 | 分类于 前端
跨域的几种方法

为什么要跨域？

不同的网站有时需要相互通信，在通信之前，需要先解决同源政策对不同网站间通信的限制。

为什么要有同源政策？

为了保证用户信息安全。

举例：

  • 一个用户同时在浏览器打开 A 和 B 两个网站，如果 B 网站可以通过脚本去访问、修改 A 网站页面的内容，恶意诱导用户执行一些操作（如转账等），但这其实并非出于用户本意，也不是 A 网站希望用户进行的操作，那么这就间接相当于 A 网站损害了用户的利益。
  • 由于现在广泛使用 cookie 来管理用户的登录状态。假设用户登录 A 网站后，没有退出。如果 B 网站可以通过脚本去读取 A 网站保存在用户浏览器里的 cookie，那么 B 网站就可以利用读取到的 cookie 来冒充用户。这也是 CSRF（Cross-Site Request Forgery，跨站请求伪造）攻击的原因之一。

因此，必须采取措施来禁止不同的网站相互通信。该措施就是同源政策。

什么是同源政策？

协议、域名、端口均相同。

以 http://www.example.com 和 https://www.example.com 为例：

  • 协议：即通信协议。如：http、https。
  • 域名：.com 是一级（也称顶级）域名，example.com 是二级域名，www.example.com 是三级域名，以此类推。
    • example.com 和 www.example.com 是两个不同的域名，故受到同源政策的限制。
  • 端口：
    • http 默认 80 端口
    • https 默认 443 端口

同源策略限制哪些行为？

  • 无法读取非同源网页的 Cookie、LocalStorage 和 IndexedDB。
  • 无法接触非同源网页的 DOM。（防止恶意修改页面内容）
  • 无法向非同源地址发送 AJAX 请求（请求会正常发送出去，服务器会正常返回响应，但浏览器会拒绝接受响应）。

跨域的方法

  1. JSONP
  2. CORS(Cross-Origin Resource Sharing)
  3. postMessage
  4. document.domain
