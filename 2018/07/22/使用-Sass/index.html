使用 Sass

发表于 2018-07-22 | 分类于 前端
Sass 使用记录

安装

  1. 使用 Sass 需要先安装 Ruby。
  2. 也可以选择用 node-sass，使用 node-sass 不要求先安装 Ruby。

使用

  1. Sass 与 Scss 的语法存在些许差异，使用时需要注意。
  2. Sass 用缩进来表示层级嵌套。像 Python 一样。另外，使用 & 在嵌套中来引用父元素，如 a:hover，可以写成 &:hover ，&:hover 需要嵌套到 a 下。
  3. Sass 不用大括号。
  4. Sass 不用在每一行样式后加上分号。
  5. 在 Sass 中，仍可以使用媒体查询语句 @media，像在 CSS 中一样。
  6. 以下划线命名开头的文件（如：_partial.sass），不会被输出，可以通过 @import 导入其他 .sass 文件。
  7. 在 .sass 文件开头通过 @import 导入其他文件：@import "partial"。可以看到，文件名开头可以不用加下划线，也不用加后缀 .sass。但为方便追踪查看导入的相关文件，推荐保持文件名一致，同时，将相对路径明确写全，即 @import "./_partial.sass"，而不只是 @import "_partial.sass"。
  8. 可以在 .sass 文件开头，通过 @import 引入普通的 .css 文件。如：@import "pathname/filename.css"。这一句经过转换，在由 .sass 文件生成的 .css 文件中，是 @import url(pathname/filename.css);。

  9. 编译

  10. 监听文件（夹）的变动

  11. 变量以 $ 开头，通过 #{$变量名} 可以镶嵌到字符串中。

  12. 混合通过 @mixin 定义，使用时通过 @include 调用。@mixin 可以传入默认参数。
  13. 继承允许在一个选择器上应用另一个选择器的样式。通过 @extend 使用继承。
  14. 函数通过 @function 定义。
  15. 注释
    1. /* 注释 */ 风格的注释会被保留到生成的 .css 文件中。
    2. // 注释 风格的注释不会被保留到生成的 .css 文件中。
