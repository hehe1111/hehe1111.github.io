cookie & session & localStorage & sessionStorage

发表于 2018-04-01 | 更新于 2018-07-21 | 分类于 前端
浏览器端存储

cookie 和 session 的区别

  1. cookie 存储在客户端。cookie 可以用来记录用户信息（一些不敏感的信息）。客户端每次向相同域名发起请求时，客户端（一般是浏览器）会自动带上相应域名的 cookie。
  2. session 一般存储在服务器端的一张表中。在表中，服务器端用不同的 sessionID（一般是随机数） 来标记不同的用户，追踪用户状态（如购物车中的商品）。
  3. session 一般是基于 cookie 实现的。服务器将 sessionID 写入 cookie 中，随 cookie 由服务器端发送到客户端。当客户端发起请求时，服务器从请求附带的 cookie 中读取 sessionID，与服务器存有的表进行比对，获取用户信息。

cookie 和 localStorage 的区别

  1. cookie 是由服务器端下发到客户端的一串字符串。客户端需要在一段时间内保存 cookie 。客户端（一般是浏览器）每次向相同域名发起请求时，都会自动带上 cookie。
  2. localStorage 跟 HTTP 无关。客户端发起请求不会带上 localStorage 中存储的值。
  3. 每个域名的 cookie 有数量限制，每条 cookie 的大小一般在 4Kb 以内。而每个域名的 localStorage 最大存储量为 5Mb左右（不同浏览器不同）。
  4. cookie 一般默认在用户关闭浏览器后就失效（浏览器不再保存该 cookie ），但是也可以通过 expires 或 max-age 字段来任意指定 cookie 的过期时间。localStorage 只要用户不手动清除缓存，则永久有效。
  5. cookie 是一串由键值对组成的字符串， localStorage 是由键值对（键值对均以字符串形式保存）组成的一张哈希表（ hash ）。

localStorage 和 sessionStorage 的区别

  • 两者仅保存期限长短不同。localStorage 永久有效，sessionStorage 则在会话结束（通常是该窗口关闭），数据会被清空。

cookie 如何设置过期时间？如何删除 cookie？

  1. 设置过期时间：
    • expires ：指定一个具体的到期时间，到了指定时间以后，浏览器就不再保留这个 cookie 。它的值是 UTC 格式。
    • max-age ：指定从现在开始 cookie 存在的秒数，比如60 * 60 * 24 * 365（即一年）。过了这个时间以后，浏览器就不再保留这个 cookie。
    • 如果同时指定了 expires 和 max-age ，那么 max-age 的值将优先生效。
    • 如果没有指定 expires 或 max-age 属性，那么这个 cookie 就是 session cookie，即它只在本次对话存在，一旦用户关闭浏览器，浏览器就不会再保留这个 cookie 。
  2. 删除 cookie的唯一方法：
    • 设置它的 expires 属性为一个过去的日期。

Cache-Control: max-age=1000 缓存 与 ETag 的「缓存」有什么区别？

  1. Cache-Control: max-age=1000 缓存：在缓存开始存在的 1000 秒内，浏览器需要该资源时，都不会向服务器发起请求，而是从缓存中直接获取。
  2. ETag 的「缓存」：一定会发起请求。请求头中会多出 If-Not-Match 字段。如果 If-Not-Match 字段与响应头中的 ETag 字段的值不同，则浏览器就会从服务器下载获取最新资源。如果相同，则是从缓存中获取。
