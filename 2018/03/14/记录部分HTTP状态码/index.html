记录部分HTTP状态码

发表于 2018-03-14 | 分类于 前端
| 字数统计: 1,000
仅记录一些常用的

1XX - 信息性状态码

  • 100 Continue(继续) - 客户端应当继续发送请求。
  • 101 Switching Protocol(切换协议) - 服务器已经理解了客户端的请求，并将通过 Upgrade 消息头通知客户端采用不同的协议来完成这个请求。

2XX - 成功状态码

  • 200 OK(成功) - 请求成功。成功的意义根据请求所使用的方法不同而不同。
  • 204 No Content(无内容) - 该响应没有响应内容，只有响应头。用户代理可以根据新的响应头来更新对应资源的缓存信息。一般在只需要从客户端往服务器发送信息，而对客服端不需要发送新信息内容的情况下使用。
  • 206 Partial Content(部分内容) - 该状态码表示客户端进行了范围请求，而服务器成功执行了这一部分的 GET 请求。响应报文中包含由 Content-Range 指定范围的实体内容。

3XX - 重定向状态码

  • 301 Moved Permanently(永久移动) - 所请求的资源已经被分配了新的 URI，新的 URI 会在响应的 Location: 头字段里找到。
  • 302 Found(临时移动) - 所请求的资源已经被分配了新的 URI，但这只是临时性质的，资源对应的 URI 将来还可能继续改变。因此客户端在以后访问时还得继续使用该 URI。新的 URI 会在响应的 Location: 头字段里找到。
  • 303 See Other(查看其他位置) - 所请求的资源存在着另一个 URI，服务器发送该响应用来引导客户端使用 GET 方法访问另外一个 URI。303 与 302 有着相同的功能，但 303 明确表示客户端应当使用 GET 方法获取资源。
  • 304 Not Modified(未修改) - 所请求的资源距离上次访问并没有变化。客户端可以直接从浏览器缓存里获取该资源。304 虽然被划分在 3XX 类别中，但是和重定向没有关系。
  • 307 Temporary Redirect(临时重定向) - 307 和 302 有着相同的含义，但是 307 会遵照浏览器标准，不会从 POST 变成 GET。

4XX - 客户端错误状态码

  • 400 Bad Request(错误请求) - 发送的请求中存在语法错误，服务器无法正常读取。需要修改请求的内容后再次发送请求。浏览器会像 200 OK 一样对待该状态码。
  • 401 Unauthorized(未授权) - 需要身份验证后才能获取所请求的资源，类似于403错误。不同点是，当浏览器初次接受到 401 响应后，会弹出认证用的对话窗口，只要正确输入帐号密码，验证即可通过。
  • 403 Forbidden(禁止访问) - 请求资源的访问被服务器拒绝了，服务器端没有必要给出拒绝的详细理由。
  • 404 Not Found(未找到) - 服务器找不到所请求的资源。
  • 408 Request Timeout(请求超时) - 客户端没有在服务器预备等待的时间内完成一个请求的发送。这意味着服务器将会切断和客户端的连接。
  • 410 Gone(已失效) - 所请求的资源已经被删除。

5XX - 服务器错误状态码

  • 500 Internal Server Error(内部服务器错误) - 服务器在执行请求时发生错误，也有可能是 Web 应用存在 bug 或某些临时的故障。
  • 502 Bad Gateway(网关错误) - 服务器作为网关且从上游服务器获取到了一个无效的HTTP响应。
  • 503 Service Unavailable(服务不可用) - 由于临时的服务器维护或者过载，服务器当前无法处理请求。这个状况是临时的，并且将在一段时间以后恢复。如果能够预计延迟时间，那么响应中可以包含一个 Retry-After: 头用以标明这个延迟时间。
