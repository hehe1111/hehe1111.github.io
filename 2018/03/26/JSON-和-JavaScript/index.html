JSON 和 JavaScript

发表于 2018-03-26 | 分类于 前端
| 字数统计: 396
两者的关系和区别

关系

JSON 是 Javascript 的一个子集：JSON 格式是 1999 年《JavaScript Programming Language, Standard ECMA-262 3rd Edition》的子集合，但 JSON 是独立于语言的文本格式。

区别

  1. JSON 是一种轻量级的数据交换格式

  2. JavaScript 是一门编程语言

  3. JavaScript 有七种数据类型：number string boolean null undefined object symbol 。
    其中 object 是广义上的对象，又可以细分为 array function 和狭义上的 object。
    狭义上的 object 由一对花括号包裹其中的键值对，
    其中，键名只能是字符串类型（string），键值则可以是以上七种数据类型的任何一种，
    键名和键值之间用 : （英文冒号）相隔，
    键名和键值组成一个键值对，键值对之间用 , （英文逗号）相隔

  4. JSON 建构于两种结构：“名称/值”对的集合和值的有序列表

    1. JSON 中的值可以是 number string object array 四种类型值
      或是 true false null 三个特殊值中的一种。
      JSON 中的值没有 undefined symbol function。
      JSON 中的 string 类型值必须用双引号括起来，
      JavaScript 中的 string 类型值则可以用单引号，也可用双引号

    2. “名称/值”对的集合与 JavaScript 中狭义的 object 有些类似。
      也是一对花括号包裹其中的键值对，
      键名也只能是字符串类型（string），但必须用双引号括起来。
      键名和键值之间用 : （英文冒号）相隔，
      键名和键值组成一个键值对，键值对之间用 , （英文逗号）相隔。
