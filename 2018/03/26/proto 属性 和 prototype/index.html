__proto__ 和 prototype

发表于 2018-03-26 | 更新于 2018-07-20 | 分类于 前端
原型对象、原型链

原型对象的作用，就是定义所有实例对象共享的属性和方法。

原型链的尽头就是 null

null 没有任何属性和方法（函数属性），也没有自己的原型

每一个函数都有 prototype 属性，指向 函数.prototype 对象，该对象有一个 constructor 属性，constructor 属性会回指函数本身，即：

constructor属性定义在prototype对象上面，意味着可以被所有实例对象继承。

constructor属性的作用是，可以得知某个实例对象，到底是哪一个构造函数产生的。

有了constructor属性，可以从一个实例对象新建另一个实例。

constructor属性表示原型对象与构造函数之间的关联关系，如果修改了原型对象，一般会同时修改constructor属性，防止引用的时候出错。

  • 比起 Object.setPrototypeOf() ，建议优先使用 Object.create()
