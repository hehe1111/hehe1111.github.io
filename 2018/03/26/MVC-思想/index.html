MVC 思想

发表于 2018-03-26 | 分类于 前端
| 字数统计: 289
几句话简介 MVC，真的就是几句话。

目的：将复杂的代码按功能分块化，使代码结构清晰，易于阅读。

  • M：Model 模型，专门负责数据相关的工作。在接收到控制器的要求后，向服务器发起请求。同时还负责接收服务器返回的响应，并提取响应中的数据，再返回给控制器。

  • V：View 视图，即用户可以看见的页面元素。用户在视图上的操作，会触发事件，控制器由此监听到用户的操作，做出相对应的反应，如局部刷新页面内容等。

  • C：Controller 控制器，监听用户在视图上的操作（如点击、滚动等事件），根据用户的操作，控制页面元素的变化，也就是控制视图。控制器同时也负责与模型进行交互，从模型获取数据，用于更新视图等，或者从视图获取用户的输入，将用户的数据传送给模型。
