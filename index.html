绑定this

发表于 2018-03-14 | 分类于 Javascript
  • call() 和 apply() 和 bind() 都可以用来指定函数内部 this 的指向，这三个方法的第一个参数都是 this 要指向的对象，如果不传入第一个参数或是第一个参数是 null 或 undefined，则 this 会指向全局对象。
  • call() 和 apply() 都会立即执行，bind() 只是返回一个具有固定 this 值的函数，可以在需要时再调用该函数。
  • bind() 每运行一次，就会返回一个新函数。
  • call() / bind() 的参数从第二个开始，都是一个一个传进去的；apply() 的第二个参数则必须是一个数组，数组中的元素会依次传进函数里。
  • bind() 可以用于为原函数的参数绑定默认值。

为 GitHub Pages 绑定域名

发表于 2017-11-27 | 分类于 配置

在已准备好 GitHub 账号 + hexo ，开启了 GitHub Pages 预览功能的情况下，为 GitHub Pages 绑定域名。

步骤：

  1. 在https://www.namesilo.com/上购买域名，购买前要先注册账号。

    1. 购买完了之后，点击 Manage My Domains 选项

    2. 勾选域名前的小方框

      1. 在 Actions for Selected Domains 一栏选中 Change Nameservers 。将 NameServer 1 的值改为 f1g1ns1.dnspod.net ； NameServer 2 的值改为 f1g1ns2.dnspod.net 。

      2. 提交。

      3. 回到“勾选域名前的小方框”这一步，选中后，在 options 一栏里选中一个蓝色小图标（鼠标悬停在其上方会提示“Manage DNS for this domain”）；

      4. 删除默认的 A 记录和 CNAME 记录。

  2. 注册 DNSpod。之后

    1. 添加域名。会自动导入域名信息，点击确认即可。

    2. 把除了记录类型为 NS 之外的记录都删掉。

    3. 点击左上方的 “添加记录” 绿色按钮。新建三条记录：

主机类型 记录类型 线路类型 记录值
@ A 默认 192.30.252.153
@ A 默认 192.30.252.154
www CNAME 默认 username.github.io(username 用 github 账户名替换)

最后，在 hexo/source/ 文件夹下新建一个文件，写入在第一步时购买的域名，格式为 xxx.com 或是 xxx.xyz 或是 xxx.me 或是 xxx.info 等。文件名为 CNMAE。

运行 hexo g -d 命令部署到 Github 上。

HTML 标签知识补充

发表于 2017-10-15 | 分类于 HTML

Bold vs. Strong

在 <strong> MDN 英文版中，一开始关于 strong 标签是这么几句话：

The HTML <strong> element gives text strong importance, and is typically displayed in bold.

Perhaps instead of rendering some text as bold you want to render it red, or a different size, or underlined, or whatever. It makes more sense to change the presentational properties of strong than it does bold.

It is important to note that <b></b> does have other uses, when one wants to draw attention without increasing importance.

所以，strong 标签应该是重在强调文本的重要性，只是恰好默认表现为加粗而已，当然也可以通过更改样式来改变该标签的呈现。而 b 标签只是单纯的加粗文本而已，并未强调文本的重要性。

通过参看<strong> MDN 中文版辅助理解，英语渣，全英 hold 不住。。。

Emphasis vs. Strong

While Emphasis is used to change the meaning of a sentence (“I love carrots” vs. “I love carrots”), Strong is used to give portions of a sentence added importance (e.g., “Warning! This is very dangerous.”)

所以，em 标签重在改变一个句子的意思（强调 love，而不再只是单纯地陈述一个事实），而 strong 标签重在强调部分句子（“Warning!” 和 “very dangerous”）的重要性。

col、colgroup 标签

两者均为空标签，不包含内容，故没有结束标签。

<col> 标签:

HTML col 元素定义表格中的列，并用于定义所有公共单元格上的公共语义。它通常位于 colgroup 元素内。

<colgroup> 标签:

HTML 中的 表格列组（Column Group） 标签用来定义表中的一组列表。

HTML <col> 标签:

<col> 标签规定了 <colgroup> 元素内部的每一列的列属性。通过使用 <col> 标签，可以向整个列应用样式，而不需要重复为每个单元格或每一行设置样式。

参看<table> 标签中的示例，用代码了解 <col>、<colgroup> 标签的用法。

替换与非替换元素

替换元素（replaced element）是指用来替换元素内容的部分并非由文档内容直接表示。–《CSS 权威指南》

也就是说大部分空标签就是替换元素。如：

a 标签，如果 src 属性不指向一个外部内容，那么元素就没有任何意义。input 元素也类似，要由一个单选框、复选框或文本输入框替换。–《CSS 权威指南》

非替换元素（nonreplaced element）：

其内容由用户代理（通常是指一个浏览器）在元素本身生成的框中显示。–《CSS 权威指南》

可参看MDN：可替换元素

滑动条：

用于输入不精确值控件。如果未指定相应的属性，控件使用如下缺省值：
min：0
max：100
value：min + (max-min)/2，或当 max 小于 min 时使用 min
step：1

以上参看自<input> 标签

具体用法可参看<input type=”range”>
